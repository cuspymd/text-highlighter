# Pages List 모두 펼치기/접기 버튼 요구사항

## 목적

`pages-list.html` 화면에서 현재 표시 중인 페이지 목록의 하이라이트 상세 영역을 한 번에 펼치거나 접을 수 있는 툴바 버튼을 제공한다.

버튼은 검색창 오른쪽 툴바 영역의 정렬 버튼 바로 오른쪽에 위치한다. `내보내기`와 `가져오기`는 별도의 `...` 더보기 메뉴에 배치하고, 더보기 버튼은 툴바의 가장 오른쪽에 둔다.

## 동작 기준

모두 펼치기/접기 버튼은 항상 “현재 화면에 표시된 페이지 목록”을 기준으로 동작한다.

- 검색어가 없으면 전체 페이지 목록이 기준이다.
- 검색어가 있으면 필터링된 검색 결과 목록이 기준이다.
- 정렬 상태는 현재 표시 순서만 바꾸며, 버튼의 동작 범위에는 영향을 주지 않는다.

## 버튼 상태 모델

버튼은 표시 중인 페이지들의 상세 영역 상태에 따라 다음 3가지 상태를 가진다.

### 1. 모두 접힘

조건:
- 표시 중인 페이지가 1개 이상이다.
- 펼쳐진 `.page-highlights` 영역이 0개다.

버튼 표시/상태:
- 의미: `모두 펼치기`
- `aria-pressed="false"`
- 클릭 시 표시 중인 모든 페이지 상세 영역을 펼친다.

### 2. 일부 펼쳐짐

조건:
- 표시 중인 페이지가 1개 이상이다.
- 일부 페이지의 상세 영역만 펼쳐져 있다.

버튼 표시/상태:
- 의미: `모두 펼치기`
- `aria-pressed="false"`
- 클릭 시 아직 접힌 페이지를 포함해 표시 중인 모든 페이지 상세 영역을 펼친다.

이 상태에서 `모두 접기`를 표시하지 않는다. 일부만 펼쳐진 상태에서 버튼의 다음 동작은 “나머지도 모두 펼치기”가 가장 예측 가능하기 때문이다.

### 3. 모두 펼쳐짐

조건:
- 표시 중인 페이지가 1개 이상이다.
- 표시 중인 모든 페이지의 상세 영역이 펼쳐져 있다.

버튼 표시/상태:
- 의미: `모두 접기`
- `aria-pressed="true"`
- 클릭 시 표시 중인 모든 페이지 상세 영역을 접는다.

## 빈 목록 상태

조건:
- 표시 중인 페이지가 0개다.

버튼 표시/상태:
- 버튼은 비활성화한다.
- `aria-pressed="false"`로 둔다.
- 클릭해도 아무 동작을 하지 않는다.

## 개별 상세 버튼과의 상호작용

각 페이지 카드의 `상세 보기/접기` 버튼을 누른 뒤에는 상단 모두 펼치기/접기 버튼 상태를 다시 계산한다.

예시:

1. 모든 페이지가 펼쳐져 있어 상단 버튼이 `모두 접기` 상태다.
2. 사용자가 페이지 하나를 개별적으로 접는다.
3. 이제 일부만 펼쳐진 상태이므로 상단 버튼은 `모두 펼치기` 상태로 바뀐다.

반대로:

1. 일부만 펼쳐진 상태에서 상단 버튼은 `모두 펼치기` 상태다.
2. 사용자가 마지막 접힌 페이지를 개별적으로 펼친다.
3. 이제 모든 표시 페이지가 펼쳐졌으므로 상단 버튼은 `모두 접기` 상태로 바뀐다.

## 검색과의 상호작용

검색어 입력 시 기존 UX처럼 검색 결과의 상세 영역은 자동으로 펼쳐진다.

- 검색 결과가 모두 자동 펼쳐진 상태이면 상단 버튼은 `모두 접기` 상태가 된다.
- 사용자가 `모두 접기`를 누르면 현재 검색 결과의 상세 영역을 모두 접는다.
- 검색어가 변경되면 새 검색 결과 기준으로 다시 렌더링되며, 검색 결과는 자동 펼침된다.
- 새로고침은 검색어와 펼침 상태를 초기화한다.

## 접근성 요구사항

- 버튼은 현재 동작을 `title`과 `aria-label`에 반영한다.
  - 모두 펼치기 상태: `Expand all highlights` / locale별 번역
  - 모두 접기 상태: `Collapse all highlights` / locale별 번역
- `aria-pressed`는 현재 표시 목록 전체가 펼쳐진 경우에만 `true`다.
- 더보기 메뉴는 다음 속성을 유지한다.
  - 더보기 버튼: `aria-haspopup="menu"`, `aria-expanded`
  - 메뉴: `role="menu"`
  - 메뉴 항목: `role="menuitem"`
- 더보기 메뉴는 Escape 키와 메뉴 외부 클릭으로 닫힌다.

## i18n 키

필요한 locale key:

- `expandAllHighlights`
  - 모두 펼치기 버튼의 title/aria-label
- `collapseAllHighlights`
  - 모두 접기 버튼의 title/aria-label
- `moreActions`
  - `...` 더보기 버튼의 title

메뉴 항목 텍스트는 기존 키를 재사용한다.

- `exportAll`
- `importHighlights`
- `exportAllTooltip`
- `importHighlightsTooltip`

## 테스트 요구사항

E2E 테스트는 다음을 검증해야 한다.

1. 초기 상태에서 상세 영역이 모두 접혀 있으면 버튼의 `aria-pressed`가 `false`다.
2. 버튼 클릭 시 표시 중인 모든 페이지 상세 영역이 펼쳐지고 `aria-pressed`가 `true`가 된다.
3. 다시 버튼 클릭 시 모든 상세 영역이 접히고 `aria-pressed`가 `false`가 된다.
4. 일부만 펼쳐진 상태에서는 `aria-pressed`가 `false`다.
5. 일부만 펼쳐진 상태에서 버튼 클릭 시 모두 펼쳐진다.
6. 개별 상세 버튼 조작으로 모든 페이지가 펼쳐지면 상단 버튼 상태가 `aria-pressed="true"`로 갱신된다.
7. 더보기 메뉴 안의 내보내기/가져오기 동작은 기존 기능을 유지한다.

## 구현 참고

상태의 원천은 별도 플래그 하나가 아니라 현재 DOM의 표시 상태를 기준으로 계산하는 것이 안전하다.

권장 helper:

- `getVisiblePageItems()`
- `areAllVisiblePagesExpanded()`
- `updateExpandCollapseAllButtonState()`
- `expandAllVisiblePages()`
- `collapseAllVisiblePages()`
- `toggleExpandCollapseAllVisiblePages()`

`expandAllActive` 같은 렌더링 보조 플래그는 “전체 펼침 렌더링을 강제할지”에만 사용하고, 버튼의 최종 표시 상태는 DOM 기준으로 재계산한다.
